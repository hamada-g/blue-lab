# Project Charter

## Project Name
Blue Lab

## Purpose
Blue Lab is a hands-on cybersecurity monitoring and incident response lab built to strengthen practical skills in systems security, log analysis, detection, alert triage, and technical documentation.

## Why This Project Exists
This project is being built to close the gap between coursework and real-world security operations experience. The goal is to create a portfolio project that demonstrates technical growth, structured thinking, troubleshooting discipline, and the ability to document security work clearly.

## Primary Goals
- Build practical experience with security monitoring tools
- Learn how to collect and review logs from real systems
- Practice creating detections for suspicious activity
- Improve scripting and automation skills
- Document technical work in a professional, portfolio-ready way

## First Milestone
Deploy a basic security monitoring environment that collects logs from one Windows system and one Linux system into a central platform.

## Current Phase 1 Scope
- Project documentation
- Home lab planning
- VM planning
- Initial security platform selection
- One Windows endpoint
- One Linux endpoint
- Centralized log collection

## Out of Scope Right Now
- Cloud integrations
- Threat intelligence feeds
- SOAR or case management platforms
- Advanced DFIR tooling
- Complex network segmentation
- Public exposure of sensitive environment details

## Success Criteria for Phase 1
- Repository is organized
- Documentation is maintained from the start
- Lab host and VM plan are defined
- One central monitoring platform is chosen
- One monitoring server and two monitored endpoints are operational
- Endpoint telemetry is being collected into the Phase 1 monitoring platform

## Guiding Principles
- Keep the setup simple and maintainable
- Document every important decision
- Build one layer at a time
- Prefer understanding over speed
- Treat the project like real engineering work

## Current Positioning
Blue Lab Phase 1 serves as the defensive baseline for a future red-lab extension. The intent is to expand from a stable monitored environment into controlled attack simulation and detection validation in later phases.
